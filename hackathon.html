    Welcome Hackers to the SITA API Platform

    Below are some quick guidelines on using the APIs to get you started. For more in-depth information on the APIs, check out the documentation under each API on the site. For those of you who are new to the world of airline and airport development and don't know your PNR from your GDS, check out our air transport Jargon page. Please click on the API links below for API keys, sample requests and data, and guidelines on using the API . The following APIs are available to hackers for this event:

    Airport API

    This API will return basic info about an airport (including lat/long and timezone), search for airport by name, get nearest airport(s) and get distances between airports.

    Bag Tag API

    This API is used to print a PDF formatted IATA compliant bag tag.

    Beacon Registry API

    This API retrieves the details of iBeacons placed at an airport.

    Boarding Pass API

    This API is used for creating and sending a boarding pass to passengers. The pass can then be upaded as needed. You must supply the passenger name and flight number in JSON data and the API will do the rest - send the Boarding Pass via email, passbook, SMS, and mobileweb. To simulate real life scenarios in the airport, you can update the boarding pass with new gate details, boarding times, departure times, etc. The API will then send an update to the passenger. Apple Wallet, PDF, email and web passes are available. IATA version 6 barcodes are supported including bag tag id and fast track indicator.

    Flight Infomation API

    This API provides flight statistics and information from airports around the world including scheduled and estimated time of arrival, aircraft type, arrival airport and marketing carrier. The service also includes notifications for status changes to a particular flight.

    Flight Follower API

    This API is a REST based service which provides map images of flights and their progress on route.

    Name Matching API

    This API is used to compare names in different formats to determine if they are the same name.

    WaitTime API

    The WaitTime API is a REST based service which provides information about the length of time that a passenger may expect to wait in a particular area, such as in airport security line, or customs queue.

    Weather API

    The Weather API is a REST based service which provides current and forecasted weather data for locations around the world.


    Click on the API links below for guidelines on using the API and sample requests and data.

    Airport API

    To access the API you will need an apiKey. Please note keys will only be available for the hackathon event!

    API Key: API-Key-To-Follow

    The following examples show you how to use the API to:

    • List all major IATA Airports
      curl -v -X GET "https://airport.api.aero/airport?user_key={insert-api-key-here}"
    • Get specific details about an airport using an IATA code
      curl -v -X GET "https://airport.api.aero/airport/AMS?user_key={insert-api-key-here}"
    • Get a list of airports matching a given name or code
      curl -v -X GET "https://airport.api.aero/airport/match/Amst?user_key={insert-api-key-here}"
    • Get the distance between two airports
      curl -v -X GET "https://airport.api.aero/airport/distance/AMS/CDG?user_key={insert-api-key-here}&units=km"
    • Find airports nearest a given latitude/longitude
      curl -v -X GET "https://airport.api.aero/airport/nearest/52.297097/4.879903?maxAirports=10&user_key={insert-api-key-here}"

    Beacon API

    There are two APIs available:

    • API to get the list of iBeacons deployed at an airport
      curl -v -X GET "https://cube.api.aero/atibeacon/beacons/1?airportCode=SITA_XS&app_id=e719dcad&app_key={insert-api-key-here}&preservePendingNewBeacons=true

    Weather API provides current weather conditions and forecast for over 1800 airports, worldwide.

    API Key: API-Key-To-Follow

    The following examples show you how to use the API to:
