    • Using the APIs

    Welcome Hackers to the SITA API Platform

    Below are some quick guidelines on using the APIs to get you started. For more in-depth information on the APIs, check out the documentation under each API on the site. For those of you who are new to the world of airline and airport development and don't know your PNR from your GDS, check out our air transport Jargon page.     Airport API

    This API will return basic info about an airport (including lat/long and timezone), search for airport by name, get nearest airport(s) and get distances between airports.

    Bag Tag API

    This API is used to print a PDF formatted IATA compliant bag tag.

    Beacon Registry API

    This API retrieves the details of iBeacons placed at an airport.

    Boarding Pass API

    This API is used for creating and sending a boarding pass to passengers. The pass can then be upaded as needed. You must supply the passenger name and flight number in JSON data and the API will do the rest - send the Boarding Pass via email, passbook, SMS, and mobileweb. To simulate real life scenarios in the airport, you can update the boarding pass with new gate details, boarding times, departure times, etc. The API will then send an update to the passenger. Apple Wallet, PDF, email and web passes are available. IATA version 6 barcodes are supported including bag tag id and fast track indicator.

    Flight Infomation API

    This API provides flight statistics and information from airports around the world including scheduled and estimated time of arrival, aircraft type, arrival airport and marketing carrier. The service also includes notifications for status changes to a particular flight.

    Flight Follower API

    This API is a REST based service which provides map images of flights and their progress on route.

    Name Matching API

    This API is used to compare names in different formats to determine if they are the same name.

    WaitTime API

    The WaitTime API is a REST based service which provides information about the length of time that a passenger may expect to wait in a particular area, such as in airport security line, or customs queue.

    Weather API

    The Weather API is a REST based service which provides current and forecasted weather data for locations around the world.

    Name Matching API

    This API allows the user to determine if the name on the boarding pass matches the name on the passport. The order and formatting of the two can be different. This API normalizes the names.

  • Boarding pass format: Smith/AllisonMs
  • Passport format: SMITH<<ALLISON<JONES

    • To access the API you will need an apiKey.

    API Key:d795ed7104f6374bfb347ad9603ccbd4

    The following examples show you how to use the API to:

    • Compare boarding pass formatted name to passport formatted name:
      curl -v -X GET -H "x-apiKey:d795ed7104f6374bfb347ad9603ccbd4" 'https://namematch-stg.api.aero/GroundServices/namematch/v3/match/XS?bpName=Smith%2FAllisonMs&mrzName=SMITH%3C%3CALLISON%3CJONES&trace=false&ruleset=SOD'

    Note: In the curl command above

  • %2F is a /
  • %3C is a greater than sign
